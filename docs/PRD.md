# PRD — 입찰 공고 분석기

> 작성일: 2026-06-19  
> 대상 팀: 선엔지니어링 수주전략팀  
> 기술 스택: Next.js (App Router) + React

---

## 1. 배경 및 목적

수주전략팀은 나라장터 입찰 공고를 수시로 검토하지만, 공고마다 분량·형식이 달라 마감일·참가자격·제출서류를 찾는 데 시간이 과도하게 소요된다. 여러 공고를 동시에 비교하거나 제출서류 준비 현황을 추적할 수 있는 도구가 없어 업무 효율이 낮다. 본 도구는 PDF 공고문 업로드만으로 핵심 정보를 즉시 정리하고, 준비 현황까지 한 화면에서 관리할 수 있게 한다.

## 3. 범위 및 제약

| 포함 | 제외 |
|------|------|
| PDF 텍스트 추출 (키워드 기반) | AI/LLM 분석 |
| 클라이언트 사이드 상태 관리 | 서버 데이터베이스 저장 |
| 문서 내보내기 | 로그인 / 회원가입 |
| 브라우저 세션 내 데이터 유지 | 나라장터 API 직접 연동 |

> **추출 정확도 주의**: PDF 레이아웃·스캔 품질에 따라 오추출이 발생할 수 있으므로, 모든 추출 결과에 "⚠ 원문 확인 필요" 경고를 항상 표시한다.

---

## 4. 기능 요구사항

### F1. PDF 업로드 및 정보 추출

- 드래그 앤 드롭 또는 파일 선택으로 PDF 업로드 (복수 파일 동시 업로드 지원)
- `pdf.js` 등 클라이언트 라이브러리로 텍스트 추출 후 키워드 패턴 매칭
- 추출 대상 필드

| 필드 | 추출 키워드 예시 |
|------|----------------|
| 공고명 | 입찰공고명, 용역명 |
| 발주기관 | 수요기관, 발주처 |
| 예정금액 | 추정금액, 기초금액 |
| 용역기간 | 계약기간, 사업기간 |
| 마감일 | 입찰마감일, 제출기한 |
| 참가자격 | 참가자격, 입찰참가자격 |
| 제출서류 | 제출서류, 입찰서류 |

- 추출 결과는 카드 UI로 표시; 각 필드 옆 연필 아이콘으로 수동 수정 가능
- 카드 상단에 "⚠ 원문 확인 필요" 배너 고정 표시

### F2. D-Day 표시

- 마감일 기준 오늘(브라우저 로컬 시간)과의 차이를 자동 계산
- 표시 규칙

| 조건 | 표시 |
|------|------|
| 8일 이상 | 파란색 `D-N` |
| 7일 이내 | 빨간색 `D-N` + 경고 뱃지 |
| 당일 | 빨간색 `D-0` |
| 마감 지남 | 회색 `마감` |

### F3. 제출서류 체크리스트 및 준비 진행률

- F1에서 추출된 제출서류 목록을 체크리스트로 자동 생성
- 항목 추가·삭제·순서 변경 가능 (비개발자 친화적 UI)
- 준비 완료 항목 수 / 전체 항목 수로 진행률(%) 계산 및 프로그레스 바 표시
- 상태값: 미시작 / 진행중 / 완료 (드롭다운 또는 토글)

### F4. 다중 공고 비교

- 기본 뷰: **대시보드 뷰** — KPI + 차트 + 사이드바에서 선택한 공고 상세 패널
- 비교 뷰: 공고 2개 이상 시 사이드바 하단 토글로 전환 가능
  - 필드별 행을 고정한 가로 테이블 비교
  - 마감일 기준 정렬 기능

### F5. 실적·경력 폼 및 문서 내보내기

- 입력 폼 항목

| 구분 | 항목 |
|------|------|
| 유사용역 실적 | 용역명, 발주처, 계약금액, 수행기간, 주요 내용 (반복 추가 가능) |
| 기술자 경력 | 성명, 직위, 보유자격, 주요 경력 (반복 추가 가능) |

- 내보내기 형식: **Word (.docx)** (docx.js 사용, 별도 설치 없이 브라우저에서 즉시 다운로드)
- 내보내기 시 공고명이 파일명에 포함됨 (예: `선엔지니어링_실적경력_OO용역.docx`)

## 6. 화면 구성 (ERP 대시보드 레이아웃)

> **2026-06-19 변경**: 단순 카드 그리드에서 ERP 대시보드 레이아웃으로 전면 개편.

### 6-1. 전체 레이아웃

```
┌──────────────┬──────────────────────────────────────────────────┐
│  선엔지니어링 │  헤더: 선택 공고명 / 오늘 날짜                   │
│  입찰 공고    ├──────────────────────────────────────────────────┤
│  분석기       │  KPI 카드 4개                                    │
│  수주전략팀   │  [전체 공고] [마감 임박] [평균 진행률] [완료 서류]│
├──────────────┼──────────────────────────────────────────────────┤
│ [+ 공고 추가] │  차트 영역                                       │
│              │  ┌─────────────────────┐ ┌─────────────┐         │
│ 공고 목록     │  │ 공고별 마감일 현황   │ │ 서류 준비   │         │
│  - 공고1 D-7 │  │ (수평 막대 차트)     │ │ 현황        │         │
│  - 공고2 D-15│  │                     │ │ (도넛 차트) │         │
│              │  └─────────────────────┘ └─────────────┘         │
│              ├──────────────────────────────────────────────────┤
│              │  선택 공고 상세 패널                              │
│              │  ⚠ 원문 확인 필요                                │
│              │  [기본정보] [체크리스트] [실적·경력]             │
│              │                                                  │
│ [대시보드/비교]│                                                  │
└──────────────┴──────────────────────────────────────────────────┘
```

### 6-2. 사이드바

- 브랜드명(선엔지니어링) + 서비스명 + 팀명 표시
- "+ 공고 추가" 업로드 버튼
- 공고 목록: 공고명(truncate), D-Day 뱃지, 체크리스트 진행률 바 인라인 표시
- 공고 클릭 → 우측 상세 패널 전환
- 공고 2개 이상일 때 하단에 [대시보드 / 비교 뷰] 전환 토글 표시

### 6-3. KPI 카드 (4종)

| 카드 | 표시 값 | 서브텍스트 |
|------|---------|-----------|
| 전체 공고 | 전체 건수 | 진행중 N건 · 마감 N건 |
| 마감 임박 | 7일 이내 건수 | 7일 이내 마감 건수 (긴급 시 빨간색) |
| 평균 진행률 | 전체 평균 % | 완료 N / 전체 N 서류 |
| 완료 서류 | 완료 항목 수 | 전체 N건 중 |

### 6-4. 차트

| 차트 | 종류 | 내용 |
|------|------|------|
| 공고별 마감일 현황 | 수평 막대 차트 (recharts) | 공고별 남은 일수. 빨강(≤7일) / 파랑(≥8일) / 회색(마감) |
| 서류 준비 현황 | 도넛 차트 (recharts) | 전체 체크리스트 미시작 / 진행중 / 완료 비율 |

## 7. 기술 구현 방향

- **PDF 파싱**: `pdfjs-dist` (클라이언트 사이드, 서버 전송 없음)
- **상태관리**: React `useState` + `localStorage` 동기화
- **체크리스트**: 상태 토글 방식 (미시작 → 진행중 → 완료)
- **Word 내보내기**: `docx` (docx.js)
- **차트**: `recharts` (BarChart, PieChart) — 클라이언트 컴포넌트에서 동적 렌더링
- **레이아웃**: ERP 대시보드 — 고정 사이드바(w-60) + 스크롤 가능한 메인 패널
- **스타일**: Tailwind CSS v4 (`@import "tailwindcss"`, `tailwind.config.js` 없음)
